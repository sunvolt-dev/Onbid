# ADR 작성 스킬

## 언제 ADR을 작성하나
- 새 라이브러리/프레임워크 선택 시
- DB 스키마 구조 변경 시
- 에러 처리 전략 변경 시
- 기존 방식을 버리고 새 방식 채택 시
- 단순 버그 수정이나 리팩토링은 ADR 불필요

## 저장 위치
docs/adr/NNN-제목.md
NNN은 기존 최고 번호 + 1

## 작성 형식
# ADR-NNN: 제목

## 상태
결정됨 (YYYY-MM-DD)

## 맥락
왜 이 결정이 필요했나

## 결정
무엇을 선택했나

## 이유
왜 이걸 선택했나 (버린 대안도 포함)

## 결과
이 결정으로 생기는 트레이드오프

## 작성 규칙
- 작업 시작 전에 작성
- 2~3문단 이내로 간결하게
- "더 좋아 보여서"가 아니라 구체적 근거 기술
