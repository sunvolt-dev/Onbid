# 온비드 공매 대시보드 시스템 설계 문서

---

## 1. 프로젝트 개요

한국자산관리공사 온비드(Onbid) 공공데이터 API를 활용해 부동산 공매 물건을 자동 수집하고, 투자 판단에 유용한 정보를 대시보드로 제공하는 시스템.

**수집 대상 카테고리:**
- 상가용및업무용건물 > 업무시설
- 용도복합용건물 > 주/상용건물
- 용도복합용건물 > 오피스텔

**수집 조건 (API 파라미터):**
- 처분방식: 매각 (`dspsMthodCd=0001`)
- 입찰방식: 인터넷 입찰 (`bidDivCd=0001`)
- 건물면적: 30㎡ ~ 200㎡

**수집 대상 지역 (Python 필터링):**
- 서울특별시, 경기도, 인천광역시
- 부산광역시, 대구광역시, 광주광역시, 대전광역시, 울산광역시
- 세종특별자치시

> API의 `lctnSdnm`은 단일값만 지원해서 지역별 API 호출을 여러 번 하는 대신,
> 전체 수집 후 Python의 `TARGET_REGIONS` 셋으로 필터링한다.

## 4. API 구조

온비드가 제공하는 API는 3종류이며, 용도에 따라 호출 시점이 다르다.

### 4.1 목록 API — 자동 수집 (백그라운드)

```
OnbidRlstListSrvc / getRlstCltrList
```

- `onbid_list_collector.py`에서 주기적으로 자동 호출
- 결과를 SQLite `BID_ITEMS`에 저장
- 한 번 호출로 최대 100건, 페이지네이션으로 전체 수집

### 4.2 상세 API — 실시간 호출 (사용자 요청 시)

```
OnbidRlstDtlSrvc / getRlstDtlInf
```

- 사용자가 물건을 클릭할 때 Flask 백엔드에서 실시간 호출
- DB에 저장하지 않고 응답을 그대로 프론트에 전달
- 필수 파라미터: `cltrMngNo` (물건관리번호), `pbctCdtnNo` (공매조건번호)
- 제공 정보: 임대차 정보, 등기사항, 점유관계, 배분요구, 사진/위치도 URL

### 4.3 입찰정보 API — 실시간 호출 (사용자 요청 시)

```
OnbidCltrBidDtlSrvc / getCltrBidInf
```

- 상세 조회와 동시에 호출
- 회차별 입찰 일정, 최저입찰가, 이전 유찰 이력 제공

### 4.4 API 호출 비용 비교

```
목록만 수집:  3그룹 × N페이지 = ~30회
상세까지 수집: 30회 + 물건수 × 2회 = ~6,000회 (물건 3,000건 기준)
```

상세 API를 수집 시마다 전부 호출하면 API 제한(10 tps)에 걸릴 수 있어서, **사용자가 클릭할 때만 실시간 호출**하는 방식을 채택.

## 5. 전체 데이터 흐름

```
[매일 자동 - cron]
온비드 목록 API
    → UPSERT → BID_ITEMS (SQLite)
    → 사라진 물건 → status = 'closed'
    → COLLECTION_LOG 기록

[사용자 행동]
대시보드 목록 (DB 조회)
    → 물건 클릭
        → Flask → 온비드 상세 API 실시간 호출 → 화면 표시
        → Flask → 온비드 입찰 API 실시간 호출 → 화면 표시
    → 북마크 클릭
        → Flask → DB is_bookmarked = 1 업데이트
```

## 6. Flask API 엔드포인트

| 메서드 | 경로 | 설명 | 데이터 출처 |
|---|---|---|---|
| GET | `/api/items` | 목록 조회 (필터/정렬 포함) | DB |
| GET | `/api/items/<id>/detail` | 물건 상세 정보 | 온비드 상세 API 실시간 |
| GET | `/api/items/<id>/bid` | 입찰 회차 정보 | 온비드 입찰 API 실시간 |
| POST | `/api/items/<id>/bookmark` | 북마크 토글 | DB |

### GET /api/items 쿼리 파라미터

| 파라미터 | 설명 | 예시 |
|---|---|---|
| `ratio_max` | 감정가 대비 비율 상한 (%) | `75` |
| `usbd_min` | 최소 유찰 횟수 | `1` |
| `sd_nm` | 시도 필터 | `서울특별시` |
| `bookmarked` | 북마크만 조회 | `true` |
| `limit` | 결과 수 | `100` |

## 7. DB 설계

### BID_ITEMS

물건 목록 수집 결과. 핵심 테이블.

| 컬럼 | 타입 | 설명 |
|---|---|---|
| `cltr_mng_no` | TEXT PK | 물건관리번호 — 온비드 고유 식별자 |
| `pbct_cdtn_no` | INTEGER | 공매조건번호 — 상세/입찰 API 호출 시 필요 |
| `onbid_cltr_nm` | TEXT | 물건명 |
| `prpt_div_nm` | TEXT | 재산유형 (압류재산 / 기타일반재산 등) |
| `cltr_usg_mcls_nm` | TEXT | 용도 중분류 |
| `cltr_usg_scls_nm` | TEXT | 용도 소분류 |
| `lctn_sd_nm` | TEXT | 소재지 시도 |
| `lctn_sggn_nm` | TEXT | 소재지 시군구 |
| `lctn_emd_nm` | TEXT | 소재지 읍면동 |
| `land_sqms` | REAL | 토지면적 (㎡) |
| `bld_sqms` | REAL | 건물면적 (㎡) |
| `apsl_evl_amt` | INTEGER | 감정평가금액 (원) |
| `lowst_bid_prc` | INTEGER | 현재 회차 최저입찰가 (원) |
| `ratio_pct` | REAL | 감정가 대비 최저입찰가 비율 (%) — 핵심 투자 지표 |
| `frst_ratio_pct` | REAL | 최초 최저입찰가 대비 현재 비율 (%) — 하락폭 파악용 |
| `usbd_nft` | INTEGER | 유찰 횟수 |
| `pbct_nsq` | TEXT | 현재 공매 회차 |
| `pvct_trgt_yn` | TEXT | 수의계약 가능여부 (Y/N) |
| `batc_bid_yn` | TEXT | 일괄입찰 여부 (Y: 여러 물건 묶음) |
| `alc_yn` | TEXT | 지분물건 여부 (Y: 일부 지분만 매각) |
| `crtn_yn` | TEXT | 정정 이력 여부 (Y: 상세 조회 시 확인 권장) |
| `rqst_org_nm` | TEXT | 공고기관명 |
| `exct_org_nm` | TEXT | 집행기관명 |
| `cltr_bid_bgng_dt` | TEXT | 입찰 시작일시 (YYYY-MM-DD HH:MM) |
| `cltr_bid_end_dt` | TEXT | 입찰 마감일시 (YYYY-MM-DD HH:MM) |
| `thnl_img_url` | TEXT | 썸네일 이미지 URL |
| `status` | TEXT | active: 진행중 / closed: 낙찰·취소로 종료 |
| `is_bookmarked` | INTEGER | 관심목록 등록 여부 (0/1) |
| `first_collected_at` | TEXT | 최초 수집 일시 (INSERT 시만 기록) |
| `collected_at` | TEXT | 마지막 수집 일시 (매 수집마다 갱신) |

**인덱스:**
```sql
idx_ratio   ON BID_ITEMS (ratio_pct)
idx_end_dt  ON BID_ITEMS (cltr_bid_end_dt)
idx_region  ON BID_ITEMS (lctn_sd_nm, lctn_sggn_nm)
idx_usbd    ON BID_ITEMS (usbd_nft)
```

### COLLECTION_LOG

수집 실행 이력 기록.

| 컬럼 | 설명 |
|---|---|
| `id` | 자동증가 PK |
| `run_at` | 수집 실행 일시 |
| `query_label` | 수집 그룹 레이블 |
| `total_count` | 수집된 전체 건수 |
| `new_count` | 신규 저장 건수 |
| `updated_count` | 기존 업데이트 건수 |
| `status` | success / fail |
| `error_msg` | 실패 시 오류 메시지 |

### ALERT_LOG

알림 발송 이력 (중복 발송 방지).

| 컬럼 | 설명 |
|---|---|
| `id` | 자동증가 PK |
| `cltr_mng_no` | 물건관리번호 (FK → BID_ITEMS) |
| `triggered_ratio` | 알림 트리거 시점의 비율값 |
| `alert_type` | ratio / deadline / pvct |
| `sent_at` | 발송 일시 |
| `status` | success / fail / skip |

## 8. 상세 조회 시 제공 정보 (온비드 API 실시간)

DB에 저장하지 않고 클릭 시 실시간으로 보여주는 항목들.

### 물건 상세 (04 API)

- 전체 주소 (지번/도로명)
- PNU 코드 (지도 API 연동용)
- 사진 목록
- 위치도
- 인도/인수 책임
- 배분요구 종기일
- 임대차 정보 (보증금, 월세, 전입일, 확정일) — 압류재산만
- 등기사항 주요 정보 (권리종류, 설정금액) — 압류재산만
- 점유관계 (점유인, 보증금, 전입일) — 압류재산만
- 배분요구 사항 (권리자, 채권금액, 말소가능여부) — 압류재산만

### 입찰 정보 (07 API)

- 회차별 입찰 시작/마감/개찰 일시
- 현 회차 최소 입찰가
- 입찰 보증금
- 유찰 누적 횟수
- 이전 입찰 결과 이력
- 공동/대리 입찰 가능여부

## 9. 물건 상태 관리

온비드에서 낙찰되거나 취소된 물건은 API 응답에서 사라진다.
매 수집 완료 후 이번 수집에 등장하지 않은 `active` 물건을 `closed`로 마킹.
물건 자체는 삭제하지 않고 이력으로 보존.

```
수집 완료 → all_collected_ids 확인
    → DB active 물건 중 collected_ids에 없는 것 → status = 'closed'
```

## 10. 알림 종류 (AlertType)

| 타입 | 조건 | 설명 |
|---|---|---|
| `ratio` | 감정가 대비 비율 특정 임계값 이하 | 저가 매물 알림 |
| `deadline` | 입찰 마감 3일 이내 | 마감 임박 알림 |
| `pvct` | pvct_trgt_yn = 'Y' | 수의계약 전환 알림 |
