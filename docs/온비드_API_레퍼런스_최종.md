# 온비드 오픈API 서비스 레퍼런스

한국자산관리공사(캠코) 차세대 온비드 | 공매 부동산 자동 수집·분석 시스템 구축용

2026년 3월 | 내부 참고용

## 1. 프로젝트 개요

### 1.1 시스템 목적

> 온비드(한국자산관리공사 공매 플랫폼)에서 업무시설·오피스텔·주상복합 등 수익형 부동산 물건을 매일 자동 수집하고, 감정가 대비 저가 물건을 자동 분류·알림한다. 수의계약 여부와 무관하게 전체 공매 물건을 수집하며, 법인 투자자 관점에서 자격제한 조건을 자동 필터링하고, 임차인·등기 리스크를 사전 식별한다.

### 1.2 전체 API 호출 흐름

| 단계 | 국문명 | 사용처 | 설명 |
|------|--------|--------|------|
| **① OnbidRlstListSrvc** | 부동산 물건목록 | 매일 전수 수집 | 조건에 맞는 물건 전체 수집 → DB 저장. |
| **② 등급 자동 분류** | — | 알림/하이라이트 | 50~60%: 알림 발송·상단 고정 \| 60~70%: 하이라이트 \| 70% 초과: 일반. |
| **③ OnbidRlstDtlSrvc** | 부동산 물건상세 | 임차인·등기 확인 | 알림+하이라이트 물건만 선택 호출. 임차인·등기·점유관계·감정평가서 URL 수집. |
| **④ OnbidCltrBidDtlSrvc** | 물건상세 입찰정보 | 자격제한 확인 | ③과 동일 물건 연속 호출. 자격제한 조건 및 이전 회차 이력 수집. |
| **⑤ OnbidCltrBidRsltListSrvc** | 물건 입찰결과목록 | 가격 패턴 분석 | 필요 시 선택 호출. 전체 유찰 이력 및 가격 하락 주기 분석. |

> **최종 회차 파악:** API는 직접 제공하지 않음. 방법① (권장) 매일 수집 시 pbctNsq를 DB 저장 → max(pbctNsq) = 최종 회차. 방법② (보조) OnbidCltrBidDtlSrvc 의 prcnBidClgList 배열 마지막 pbctNsq + 1 로 추정.

### 2-3. 물건상세 입찰정보 조회 [자격제한·입찰조건 확인]

`https://apis.data.go.kr/B010003/OnbidCltrBidDtlSrvc`

## 5. 개발 시 주의사항

### 5.1 API 호출 최적화

> ① 목록 API: 매일 1회 전수 수집. ② 상세(②)+입찰정보(③): 알림·하이라이트 등급 물건만 선택 호출. 공공데이터포털 API는 일일 호출 한도 존재. 불필요한 호출 자제.

### 5.2 날짜 형식

### 5.4 호실·층수 파싱

> API에 호실·층수 전용 필드 없음. onbidCltrNm(물건명) 텍스트에서 정규식으로 파싱.

| 파싱 항목 | 방법 | 사용처 | 설명 |
|-----------|------|--------|------|
| **호실 추출** | `re.search(r'(\d+)호', name)` | 호실 | 예: '1303호' 추출 |
| **층수 추출** | `re.search(r'(\d+)층', name)` | 층수 | 예: '3층' 추출 |
| **층수 추정 (호수에서)** | `num[:-2] + 층` | 층수 추정 | 예: 1303호 → 13층, 503호 → 5층 |
| **건물 전체 층수** | locVntyPscdCont 파싱 또는 외부 API | 참고 | API 전용 필드 없음. 텍스트에서 힌트 확인. |
